# Kiwi Challenge Memory

## Problem Summary
- Time-dependent asymmetric TSP variant
- n areas (each with 1+ airports), start at a given airport
- Make EXACTLY n flights (days 1..n): visit n-1 other areas, return to start_area
- Path has n+1 airports: [start, ..., any_airport_in_start_area]
- Day-0 flights are available every day; multiple flights same day → use min cost
- Time limits: ≤20 areas + <50 airports → 3s; ≤100 + <200 → 5s; else 15s

## Critical Implementation Notes
- **visited starts EMPTY** (frozenset()), not {start_area}; start_area is the LAST visit
- Reserve start_area for the final flight: `if len(remaining)>1: remaining.discard(start_area)`
- Many airports are DEAD-ENDs (no outgoing flights). Filter candidates to avoid them:
  - Check `problem.airport_reachable[ap] & after_remaining` before choosing ap
  - Without this filter, rollouts fail for instances 5 & 6 (66% dead-end airports)
- Terminal: `len(visited) == n` (all n areas visited, start_area being the last)

## Solver: solver.py
- `Problem`: parses, precomputes adjacency + reachability
- `rollout()`: completes tour with "greedy"|"nn"|"mixed"|"random" strategy
- `MCTSSolver`: MCTS with UCT; best solution tracked across all rollouts
- `tune_parameters()`: tests 20 param configs, returns (best_cfg, best_cost, best_path)
- `main()`: 40% tune + 60% solve; validates with verify_solution()

## Validator: validate.py + validator.ipynb
- `validate(instance_file, path)` — path as list or space-separated string
- Notebook has 2 cells: set INSTANCE + PATH, then run
